# AutoMergePublicNodes-Optimized 2.3.0 Release Notes

## Summary

Version 2.3.0 upgrades the project from a latency-oriented node output pipeline to a configurable and observable node quality pipeline. The default behavior remains compatible: GitHub Actions still uses `config/scoring.yaml`, and real proxy testing still decides whether a node is usable. Scoring only controls the ranking of usable nodes.

## What changed

### Configurable scoring

Node ranking now uses a weighted scoring configuration instead of only sorting by latency. The default profile is:

```text
config/scoring.yaml
```

Default weights:

| Factor | Weight | Meaning |
|---|---:|---|
| `latency` | 35 | Real proxy test latency |
| `jitter` | 15 | Real-test jitter across targets |
| `tcp` | 10 | TCP pre-check latency |
| `protocol_history` | 20 | Historical protocol pass rate |
| `source_history` | 20 | Historical source pass rate |

### Scoring profiles

Additional profiles are available for different use cases:

| Profile | Use case |
|---|---|
| `config/scoring.low_latency.yaml` | Prefer the fastest nodes |
| `config/scoring.stability.yaml` | Prefer stable low-jitter nodes |
| `config/scoring.source_quality.yaml` | Prefer trusted sources and protocols |

Run locally with a profile:

```bash
python main.py --top-n 100 --test-limit 500 --scoring-rules config/scoring.stability.yaml
```

### Better observability

New or enhanced outputs:

| Output | Purpose |
|---|---|
| `output/stats.json` | Includes scoring config and Top node score breakdown |
| `output/health_report.md` | Human-readable health report with scoring details |
| `output/scoring_profiles.md` | Comparison table for scoring profiles |
| GitHub Actions Summary | Run overview, scoring weights, score breakdown, output links |
| README auto status block | Latest run status shown directly on the repository homepage |

### CI and validation

- CI validates all `config/scoring*.yaml` profiles.
- Weight sum different from 100 is a warning, not a hard failure.
- Regression tests cover scoring config, breakdown accounting, health report, scoring profile report, and Actions Summary.

## Compatibility

No subscription URL changes are required. Existing users can continue using:

```text
output/verified.txt
output/verified.yaml
output/verified.json
```

The only behavior change is ranking: usable nodes are now ordered by comprehensive quality score first, with latency as a tie-breaker.

## Recommended validation

```bash
python -m compileall -q main.py core tools tests
python tools/validate_config.py --sources config/sources.yaml --filter-rules config/filter_rules.yaml --scoring-rules 'config/scoring*.yaml'
python -m unittest -v tests.test_regressions
python tools/scoring_profiles_report.py --profiles 'config/scoring*.yaml' --output output/scoring_profiles.md
```

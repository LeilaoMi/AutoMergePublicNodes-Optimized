## 原因确认
本轮 Actions 自动产物提交 `91ee3df chore: update nodes 2026-06-06 00:43 CST` 中：
- `stats.json`: `nodes_raw=33441`, `nodes_dedup=15434`, `nodes_all_output=15434`
- `trend_history.json` 最后一轮：
  - `nodes_real_ok=38`
  - `nodes_verified_output=180`
  - `nodes_global_output=199`
  - `output_guard.verified.preserved=true`, `previous_count=180`, `proposed_count=38`, `min_retain_ratio=0.7`
  - `output_guard.global.preserved=true`, `previous_count=199`, `proposed_count=40`, `min_retain_ratio=0.7`

结论：不是 Actions 没跑；是 `main.py` 的缩水守门逻辑生效。本轮真实通过数低于上一轮 70%，所以保留了上一轮 `verified.*` / `global.*`，导致仓库真测文件看起来未更新。

## 已修改
1. `.github/workflows/update.yml`
   - 新增手动输入 `min_retain_ratio`，默认 `0`。
   - `Run pipeline` 步骤读取 `MIN_RETAIN_RATIO` 并传给：
     `python main.py --min-retain-ratio "$MIN_RETAIN_RATIO"`
   - 默认关闭缩水守门，使 GitHub Actions 写入本轮真测结果。

2. `README.md`
   - 将原“低于上一轮 70% 会保留上一轮”说明改为：GitHub Actions 默认关闭缩水守门。
   - 说明手动运行时可把 `min_retain_ratio` 设为 `0.7` 来恢复保留上一轮的稳定订阅行为。
